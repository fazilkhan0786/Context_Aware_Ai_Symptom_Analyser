Thank you for your interest in contributing to the AI Symptom Checker project! This document outlines how to contribute while maintaining proper attribution and respect for the original creator.

## Code of Conduct

By contributing to this project, you agree to:

1. **Respect Authorship**: Acknowledge Fazilkhan Malek as the original creator
2. **Follow MIT + Attribution License**: All contributions must comply with the license terms
3. **Professional Conduct**: Maintain a professional and respectful environment
4. **Honest Attribution**: Properly attribute all sources and influences

## How to Contribute

### Reporting Issues

1. Check existing issues first to avoid duplicates
2. Provide clear, detailed descriptions
3. Include steps to reproduce (for bugs)
4. Mention the environment/versions used

### Submitting Pull Requests

1. **Fork the repository**
2. **Create a feature branch**: `git checkout -b feature/your-feature-name`
3. **Make your changes** with clear, descriptive commits
4. **Include author headers** in new files (see below)
5. **Test thoroughly** before submitting
6. **Write clear PR description** explaining the changes
7. **Respect the original design** and architecture

### Code Style & Headers

All new files must include an author header:

```typescript
/**
 * AI Symptom Checker - [Component/Feature Name]
 * 
 * @author Fazilkhan Malek (original creator)
 * @contributor [Your Name] (2026)
 * @license MIT with Attribution
 * @description [Brief description]
 */
```

## Commit Message Guidelines

Follow conventional commit format:

```
type(scope): description

- feat: New feature
- fix: Bug fix
- docs: Documentation
- refactor: Code refactoring
- test: Tests
- chore: Build/tooling
```

Example:
```
feat(symptom-analyzer): add advanced filtering for chronic conditions
```

## Pull Request Process

1. Ensure your code follows the project style
2. Update relevant documentation
3. Include tests for new features
4. Reference any related issues
5. Description should be clear and concise
